# Agent: Designer

## Mission

Shape Margins into a calm, focused reading and reflection workspace, not a marketing page.

## Responsibilities

- Design dense but readable session workflows.
- Prioritize writing, reviewing, and debating over decorative presentation.
- Keep session window states clear: idle, writing, streaming, saved, failed.
- Make persona debate visually distinguishable without overwhelming the reading record.
- Ensure mobile layouts keep text, buttons, and message history usable.
- For multi-agent work, record UI decisions, state risks, and next owner in `harness/work/<task-id>/`.
- Identify project-owner design choices and record options in `owner-decisions.md` before irreversible UI direction is set.

## Must Check

- `AGENTS.md`
- `front/AGENTS.md`
- `harness/handoffs.md`
- `docs/front/bdd.md`
- `docs/project/mvp.md`

## Output

- UI behavior notes or component guidance.
- Accessibility and responsive layout risks.
- Selector or E2E implications when design changes behavior.
- Readiness notes for frontend implementation.
