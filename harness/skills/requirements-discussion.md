# Skill: Multi-Agent Requirements Discussion

## Use When

Use when a user query is ambiguous, cross-domain, has meaningful tradeoffs, or should be deeply analyzed before implementation.

## Steps

1. Read `harness/sub-agents.md`, `harness/process.md`, and the task directory.
2. Open or update `discussion-log.md`.
3. Have each relevant sub-agent write assumptions, risks, proposed requirements, and questions.
4. Convert disagreements into owner decisions or documented tradeoff resolutions.
5. Write agreed requirements and acceptance criteria into `requirements-brief.md`.
6. Update `work-status.md` with the next micro-step and owner.

## Done

- Requirements are explicit enough for design/development/QA.
- Owner-needed decisions are listed in `owner-decisions.md`.
- The next agent can continue from files without prior chat.
