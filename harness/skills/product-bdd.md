# Skill: Product BDD

## Use When

Use for feature scoping, MVP decisions, user flows, and acceptance criteria.

## Steps

1. Read `docs/project/mvp.md`.
2. Identify affected domains.
3. Update `docs/<domain>/bdd.md` with Given/When/Then scenarios.
4. Record durable product decisions in `docs/project/`.
5. Mark deferred work explicitly instead of leaving it implicit.
6. Check `harness/handoffs.md` before handing work to design or implementation.

## Done

- Behavior is testable.
- Setup, action, outcome, and persistence expectation are clear.
- The scenario can guide frontend, backend, and DB work.
